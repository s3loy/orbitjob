## 关键发现

1. **Domain 纯函数零分配**: `DecideDispatch` (13-20 ns, 0B), `Pause/Resume` (~1 ns, 0B) 已达极致
2. **Cron 解析昂贵**: `time.LoadLocation` + `cron.ParseStandard` 占 ~95% 的 `NormalizeCreate` 开销 (~100μs, 43KB)
3. **PG 操作延迟**: 写入操作 1.4-7.6ms，其中 audit INSERT 占显著比例
4. **批量操作线性**: `RefreshEffectivePriority` 从 10→1000 行，延迟 1.4→34ms（近似线性）
5. **Pipeline dispatched=0**: 调度器正常创建实例，但 dispatcher 未找到 pending 实例，需排查
